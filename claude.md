Always invoke the front-end design skill before writing any front-end code every session, no exceptions.

## Screenshot Workflow
- After writing or updating front-end code, always start the local server.
- Use Puppeteer to take a screenshot of the page (or specific sections).
- Analyze the screenshot to ensure the design is professional, branded, and matches the requirements.
- If something looks wrong, fix the code and take a new screenshot to verify.
- Save temporary screenshots in a folder called `temporary_screenshots`.
